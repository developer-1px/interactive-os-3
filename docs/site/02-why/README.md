# Why — 6편 시리즈

ds의 6가지 원칙을 한 편씩 다룬다. 각 편은 "왜 이 결정을 내렸는가" + "반례 Q&A" + "더 깊이"로 구성된다.

- [2.1 Overview](./01-overview.md) — 전체 why 개관과 6원칙 표
- [2.2 Classless](./02-classless.md) — 스타일 전용 className을 금지하는 이유
- [2.3 No Variant](./03-no-variant.md) — `variant` prop을 금지하는 이유
- [2.4 Data-Driven](./04-data-driven.md) — children JSX 대신 data prop을 받는 이유
- [2.5 De Facto Standard](./05-de-facto-standard.md) — 2곳 수렴 룰
- [2.6 Declarative Serialization](./06-declarative-serialization.md) — 메타-원칙

각 원칙은 독립적으로 읽을 수 있지만, 6편 모두 [Declarative Serialization](./06-declarative-serialization.md)이라는 메타-필터에서 도출된다. 한 편만 고른다면 그 마지막 편을 추천한다.
