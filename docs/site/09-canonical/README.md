# 9장 · Canonical

ds의 정본(canonical form)은 추상 원칙과 코드 실체 사이를 1:1로 잇는다. 이 장은 헌장(불변 원칙) → 영역별 선언 → 코드의 실체(devices) → 강제 장치(lint)의 순서로 정본을 펼친다.

## 목차

- [01 · 정본 헌장](./01-charter.md) — C1~C6 6개 불변 원칙
- [02 · 정본 선언](./02-declarations.md) — 영역별 정본·임시·유산
- [03 · Devices](./03-devices.md) — 코드에 박힌 5층 장치 트리
- [04 · Lint 1:1 매핑](./04-lint-rules.md) — 정본 1행 ↔ lint 1개
- [05 · 갱신 절차](./05-update-procedure.md) — 누가, 어떻게, 언제

## 두 축

- **헌장**(01) — 정본의 *왜*. 시간이 지나도 잘 안 바뀐다.
- **선언**(02) — 정본의 *무엇*. 영역마다 1행씩, 임시·유산까지 적는다.
- **devices**(03) — 정본의 *어디*. CANONICAL.md 1행이 코드 어느 파일·함수에 박혀 있는지.
- **lint**(04) — 정본의 *강제*. 1행이 lint 1개로 직결되지 않으면 정본은 종이 위에서만 산다.
- **절차**(05) — 정본의 *변경*. 흔들릴 때 어떻게 다시 좁히는가.
